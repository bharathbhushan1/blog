# Project: Bharath's Blog

Hugo static site (theme **PaperMod**), deployed to GitHub Pages at
`https://bharathbhushan1.github.io/blog/` (note the `/blog/` base path).
Posts live in `content/posts/`. Solo blog — see Git workflow below.

## Images — where files go

There are three image locations. Pick by **how the image is referenced**, and
never confuse them:

| Reference style | Put the source file in | Served at |
|---|---|---|
| Markdown in a post: `![alt](images/foo.svg)` | **`assets/images/`** | `/blog/images/foo.svg` |
| Hugo template/partial: `{{ "images/foo.svg" \| relURL }}` | **`static/images/`** | `/blog/images/foo.svg` |
| — (build output, do not edit) | `public/images/` is **gitignored** | regenerated by `hugo` |

Why `assets/` for post images: PaperMod's
`themes/PaperMod/layouts/_markup/render-image.html` render hook resolves the
markdown path with `resources.Get`, which reads the **`assets/`** dir, and
publishes via `.RelPermalink`. So `assets/images/foo.svg` is the *canonical,
tracked* source for post images.

- **Do not hand-edit or hand-sync `public/`.** It is gitignored build output;
  `hugo` regenerates it from `assets/` + `static/` + content. (Earlier work in
  this repo manually mirrored a copy into `public/images/` — that's unnecessary.)
- Commit only the source in `assets/images/` (or `static/images/`).

## Previewing / transforming SVGs

`rsvg-convert`, `cairosvg`, and headless Chrome are **not installed**. Use
QuickLook, and write scratch output to the gitignored **`tmp/`** dir (not `/tmp`):

```bash
qlmanage -t -s 1600 -o tmp public/images/foo.svg   # -> tmp/foo.svg.png
```

Then Read `tmp/foo.svg.png` to eyeball it. `tmp/` is in `.gitignore` precisely
so these renders never get committed.

To verify a content/image change actually builds and publishes:

```bash
hugo --quiet --destination tmp/buildtest
ls tmp/buildtest/images/        # confirm the image landed at /images/
```

Prefer this **one-shot** build for verification — it exits on its own. Avoid
leaving a long-running `hugo server` behind. If you do start one (or find one
running), shut it down when finished:

```bash
ps aux | grep '[h]ugo'   # find any running server (e.g. `hugo server --port 1313`)
kill <pid>               # stop it
```

## Diagram SVG style guide

Hand-author diagram SVGs (don't ship Excalidraw exports — they embed a base64
font and are huge/unreadable). Match the look of `assets/images/ai-agents.svg`:

- `viewBox` only (no fixed width/height) so it scales; ~2:1 landscape works well.
  Note: post diagrams are capped to `max-width: 600px` by CSS in
  `layouts/partials/extend_head.html` (selector `img[src$="ai-agents.svg"]`).
- System font stack: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, ...`.
- Soft pastel palette on a white figure panel: slate borders `#cbd5e1`, indigo
  `#ede9fe/#5b21b6`, blue `#dbeafe/#1d4ed8`, green `#dcfce7/#166534`, amber
  `#fef3c7/#92400e`. Rounded rects, `feDropShadow` for subtle depth.
- Arrows: a single `<marker orient="auto-start-reverse">` with both
  `marker-start` and `marker-end` for **bidirectional** flows. Make a flow
  bidirectional whenever data goes both ways (read/write, request/response).
- **Light-only** is fine — a white panel reads correctly under PaperMod's
  auto light/dark themes. No dark-adaptive variant needed.

- This is a solo blog; commits go **directly to `main`** (no feature branches) —
  every post is a direct push to `main`.
- Commit/push **only when asked**.
